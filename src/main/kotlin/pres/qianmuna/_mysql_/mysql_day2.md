

# mysql 索引 原理

    添加 索引
    alter table users_info add index xxx(filds)
    
> 做了什么？ 本质 是什么？

    索引 是一个 特殊 的 数据解构 ， 放在 磁盘中的 占地较大
    
    like 效率低？ 创建 全文 索引
    fulltext key 'xxx'(xxx)
    ....
    
> 二分 查找
    
    折半 ， 对有序 数组 查找 效率搞
    
>    二叉查找树
 
    左树 < root
    右数 > root
    最坏 情况 变成 单链表
    
>    平衡二叉查找树 （AVL）    
 
    左右 树 深度 差 绝对值 不超过1    
    超过 - 左旋 / 右旋
    节点 包含 数据引用地址 ， 左右节点指针
    //
    问题 树高 / 磁盘io 高
    
>    减低树高 -> 多叉树
 
    b树 多路平衡二叉树
    一个 度放多个 节点
    指向 下一个 度 的地址 // 去命中
    2的深度 可以 放入 千万 的数据 - -！
